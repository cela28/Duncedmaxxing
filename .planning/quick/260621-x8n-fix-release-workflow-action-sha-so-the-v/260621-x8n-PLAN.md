# Quick Task 260621-x8n: Fix release workflow action SHA

## Problem

The `package-release` job in `.github/workflows/release.yml` pins
`softprops/action-gh-release` to commit SHA
`c062e08bd532815e2082a07b400ef65ab24e279c` (commented `# v2.3.2`). That SHA does
not exist in the action's repository. The v1.0.0 release run
(`27763661229`) therefore failed at "Set up job" with:

> Unable to resolve action `softprops/action-gh-release@c062e08bd532815e2082a07b400ef65ab24e279c`, unable to find version `c062e08bd532815e2082a07b400ef65ab24e279c`

Because the job never started, no zip was built and the release has **zero
assets** — the addon cannot be downloaded or installed.

The correct commit SHA for tag `v2.3.2` (verified via the GitHub API:
`repos/softprops/action-gh-release/git/refs/tags/v2.3.2`) is
`72f2c25fcb47643c292f7107632f7a47c1df5cd8`.

## Scope

This task covers the durable in-repo fix only. Re-cutting the GitHub release
(moving the `v1.0.0` tag and recreating the release to fire `release: created`)
is an outward-facing remote operation handled separately by the orchestrator
with user confirmation.

## Tasks

### Task 1: Correct the pinned action SHA

- **files:** `.github/workflows/release.yml`
- **action:** On the `Upload release asset` step (line ~56), replace
  `softprops/action-gh-release@c062e08bd532815e2082a07b400ef65ab24e279c  # v2.3.2`
  with
  `softprops/action-gh-release@72f2c25fcb47643c292f7107632f7a47c1df5cd8  # v2.3.2`.
  Change only the 40-char SHA; keep the `# v2.3.2` comment and everything else
  identical.
- **verify:** `grep -n "action-gh-release@72f2c25fcb47643c292f7107632f7a47c1df5cd8" .github/workflows/release.yml`
  returns the line; `grep -c "c062e08" .github/workflows/release.yml` returns 0.
- **done:** The workflow references the real, resolvable v2.3.2 commit SHA so
  the `package-release` job can resolve the action on the next release run.

## Must-haves

- **Truths:** The action reference resolves to an existing commit in
  `softprops/action-gh-release`.
- **Artifacts:** `.github/workflows/release.yml` with the corrected SHA.
- **Key links:** `.github/workflows/release.yml:56`
