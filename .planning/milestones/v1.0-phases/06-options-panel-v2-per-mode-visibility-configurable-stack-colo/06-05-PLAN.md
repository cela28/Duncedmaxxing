<objective>
Close the stack-color default-display gap from Phase 6 UAT (test 3): the 4 per-stack color inputs all render as ffffff. Root cause is a format mismatch — Options.lua ColorToHex (lines 20-32) reads named keys color.r/.g/.b/.a, but DEFAULTS.tip.stackColors (Core.lua:30-35) are positional arrays, so the named keys are nil and ToByte(nil) falls back to 1.0 → ff for every channel. Fix by standardizing the defaults to named-key form and bumping SETTINGS_MIGRATION so persisted positional DBs re-seed on load.

Purpose: Users must see accurate default colors (white/green/yellow/red) in the stack pickers, matching what the tracker actually renders.
Output: Named-key DEFAULTS.tip.stackColors, a bumped SETTINGS_MIGRATION token, and updated core_spec.lua assertions. The renderer is left untouched — ColorTuple (TipOfTheSpear.lua:137-145) already reads both positional and named shapes.
