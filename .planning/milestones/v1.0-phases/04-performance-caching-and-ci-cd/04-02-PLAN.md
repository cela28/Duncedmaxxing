  <action>
    The 6 pre-existing luacheck warnings are all W212 (unused argument `self`) on functions defined with `:` method syntax that never reference `self` in their body. These functions use `:` syntax for API consistency (callers use `DMX:Print(msg)`, `Options:CanChange()`, etc.) so changing to dot syntax would break all call sites.

    The cleanest fix: add `"212/self"` to the `ignore` list in `.luacheckrc`. This suppresses W212 only for the argument named `self` — any other unused arguments are still flagged. This is the standard approach for WoW addons where `:` method syntax is used for API consistency even when `self` is not needed.

    Edit `.luacheckrc` line 42: change `ignore = { "432" }` to `ignore = { "432", "212/self" }`.

    No changes to Core.lua or Options.lua source code are needed with this approach.
  </action>

    - Triggers on: release created (per D-06) AND push to main AND pull_request to main (per D-14)
    - Steps:
      a. actions/checkout@v4
      b. Install Lua 5.1 and LuaRocks: `sudo apt-get update && sudo apt-get install -y lua5.1 liblua5.1-0-dev luarocks`

      b. Inject version into TOC (per D-10): extract tag from `github.ref_name`, strip `v` prefix via `${TAG#v}`, run `sed -i "s/^## Version: .*/## Version: $VERSION/" Duncedmaxxing/Duncedmaxxing.toc`, echo the injected version for CI log diagnostics
