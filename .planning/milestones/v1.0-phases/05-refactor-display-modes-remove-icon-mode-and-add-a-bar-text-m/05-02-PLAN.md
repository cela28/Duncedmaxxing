<objective>
Update the test suite for the two-mode world and make it runnable in this environment. Remove the icon-mode display assertions from `spec/core_spec.lua`, retain/adjust bar and number coverage, and add an assertion that a stored `"icons"` displayMode normalizes to `"bar"`.

Because this environment has NO native Lua/busted toolchain (CI uses busted on GitHub Actions; locally there is none), regression runs go through a fengari (Lua-VM-in-JS) harness. This plan adds a small committed node runner (`spec/run.cjs`) built on the npm package `fengari` that loads the existing busted-style spec files under a minimal `describe`/`it`/`before_each`/`assert.*` shim, so DISP-04 ("passes via the fengari harness") is concretely executable here.

The runner resolves fengari LOCAL-FIRST (`require.resolve('fengari')`) and only falls back to an `npx -y -p fengari@0.1.5` fetch when no locally-resolvable copy exists — so a committed fengari makes the gate work offline / cold-cache, and a missing dependency surfaces as an explicit message rather than an opaque network error. run.cjs also supports a `--self-test`/`--dry-run` smoke mode that boots the VM against a synthetic inline spec and emits a pass/fail line, so a broken or stubbed runner is caught before any spec file is edited.

PREREQUISITE: the Task 3 full-suite gate (and the Task 1 self-test) require either a committed/locally-resolvable fengari OR an npm-reachable environment. An environment with neither cannot run these gates.

Purpose: Keep the regression suite green and locally runnable after icon removal.
Output: A passing fengari run of the spec suite with icon-mode assertions removed and bar/number coverage intact.
</objective>
