## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User hex input | Color strings typed into the 4 new stack color input fields; validated by ParseHexColor (returns nil for invalid) |
| C_Timer callback | Reset Colors revert timer fires asynchronously; guarded by resetColorsPending flag check |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-03 | Tampering | Stack color hex inputs | low | accept | ParseHexColor returns nil for invalid input; setValue returns false which prevents config write and triggers RefreshTracker rollback. No network surface. |
| T-06-04 | Denial of Service | Reset Colors timer | low | accept | C_Timer.NewTimer creates at most one 3-second timer; cancelled on second click or Refresh(). No resource exhaustion path. |
| T-06-05 | Tampering | CopyDefaults reference leak | low | mitigate | Reset Colors uses CopyDefaults to deep-copy default stackColors, preventing direct reference to DEFAULTS table. Verified by the existing CopyDefaults implementation which recurses on table values. |

No network surface, no authentication, no cross-user data. Purely local UI configuration.
</threat_model>
