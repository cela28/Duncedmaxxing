# Phase 6 — UI Design Contract: Options UI Overhaul

> Visual and interaction contract for the WoW addon options window overhaul.
> Generated by gsd-ui-researcher. This project uses WoW's native frame-based widget
> system (CreateFrame, CreateTexture, CreateFontString) — not HTML/CSS/React.
> All measurements are pixel values passed directly to WoW API calls.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — WoW Widget API only |
| Preset | not applicable |
| Component library | WoW Widget API (CreateFrame, UIPanelButtonTemplate, UICheckButtonTemplate, InputBoxTemplate) |
| Icon library | WoW built-in interface textures only |
| Font | GameFontNormal (14px bold) / GameFontNormalSmall (11px normal) |

> shadcn gate: skipped — Lua/WoW addon, no web framework.

## Spacing Scale

WoW pixel spacing used throughout Options.lua. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Label-to-checkbox gap (`LEFT` +2 becomes ~4px visual gap) |
| sm | 8px | Close button inset from window edge (right edge -38, top -6) |
| md | 16px | Left margin for all controls (TOPLEFT x=16) |
| lg | 24px | Vertical step between section headers and first control |
| xl | 30px | Vertical step between consecutive inputs in a column |
| 2xl | 48px | Vertical gap from window top to mode selector row (y=-43 baseline) |
| 3xl | 64px | Vertical gap between major sections (header → mode row → first section) |

Exceptions:
- Section header label to first input beneath it: 28px (y-delta of -28 from header baseline)
- Title bar total height: 30px
- Button row bottom margin: 10px above window bottom edge

## Typography

WoW font objects, not CSS. Sizes are the point sizes passed to WoW's font system.

| Role | WoW Template | Approx Size | Weight | Usage |
|------|-------------|-------------|--------|-------|
| Window title | `GameFontNormal` | 14pt | Bold | "Duncedmaxxing" title, section headers ("Position", "Bar", "Number Mode", "Colors") |
| Control label | `GameFontNormalSmall` | 11pt | Normal | All `CreateInput` and `CreateText` field labels ("X", "Y", "Width", "Height", "Fill", etc.) |
| Button text | `UIPanelButtonTemplate` default | 12pt | Normal | All `CreateButton` instances |
| Stack number display | `numberFontSize` from config | 22pt default | Normal | Number mode tracker display (not in options window itself) |

Rules:
- Use `GameFontNormal` for every section header and the window title bar.
- Use `GameFontNormalSmall` for every `CreateText` label that precedes an input field.
- Never create a new FontString object with hard-coded pixel sizes — always reference the named WoW font template or the `numberFontSize` config value.

## Interaction Contracts

### Mode Toggle Buttons ("Bar" / "Number")

- Both buttons always rendered and always clickable (no disabled state).
- Clicking a mode button calls `Options:SetMode(mode)`, which sets `cfg.displayMode` and calls `RefreshTracker()`.
- After `SetMode`, `Options:Refresh()` is called which triggers the show/hide visibility swap and `window:SetSize(386, newHeight)`.
- No visual highlight on active mode button — text label in the header row ("Display: Bar") serves as the active mode indicator.

### Lock Toggle Button

- Source: D-04 (CONTEXT.md)
- Single button. Label reads "Unlock" when `DMX:GetDB().locked == true`. Label reads "Lock" when `DMX:GetDB().locked == false`.
- No color tinting. Text swap is the only state indicator.
- On click: toggle `DMX:GetDB().locked`, call `DMX:ForEachModule("ApplyLock")`, call `RefreshTracker()`, call `Options:Refresh()`.
- Replaces the two separate "Unlock Bar" + "Lock Bar" buttons from the current implementation.

### Reset Colors Button (number mode only)

- Source: D-05 (CONTEXT.md)
- Two-click inline confirmation. No `StaticPopup` dialog.
- State 1 (default): Button text = "Reset Colors".
- On first click: Button text changes to "Confirm Reset". A `C_Timer.After(3, revert)` is scheduled to restore text to "Reset Colors" if no second click occurs.
- State 2 (pending): Button text = "Confirm Reset".
- On second click within 3 seconds: Execute color reset — write default `STACK_COLORS` values to `db.tip.stackColors`, call `RefreshTracker()`, call `Options:Refresh()`, cancel the revert timer, reset button text to "Reset Colors".
- On timer expiry (no second click): Restore button text to "Reset Colors". No reset performed.
- Combat guard: `Options:CanChange()` check on both clicks.

### Hex Color Inputs (all color fields)

- Follows existing `CreateInput` pattern exactly.
- `OnEnterPressed` and `OnEditFocusLost` both call the Apply function.
- `OnEscapePressed` reverts the text to the current config value.
- Apply function: call `ParseHexColor(text)` → if nil, silently restore prior value (no error message) → if valid, write to config, call `RefreshTracker()`.
- Stack color inputs (0–3) write to `GetCfg().stackColors[index]` where index is 0-based.

### Numeric Inputs (Width, Height, Border, Scale, Text size, Position X/Y, Empty %)

- Same pattern as hex inputs, using `Clamp(value, min, max)`.
- If `Clamp` returns nil (non-numeric input), silently restore prior value.
- Validation ranges:
  - Width: 20–2000
  - Height: 4–200
  - Border: 0–10
  - Scale: 0.5–2
  - Text size: 8–96
  - Position X/Y: -4000–4000
  - Empty %: 0–100

### Window Drag

- Dragging is enabled outside combat only (`InCombat()` guard in `OnDragStart`).
- `OnDragStop` saves position to `cfg.optionsX` / `cfg.optionsY`.

### Combat Lockdown

- `window:OnShow`: if `InCombat()`, immediately `Hide()` and print error.
- `PLAYER_REGEN_DISABLED` event hides the window.
- Every button and input calls `Options:CanChange()` before mutating config.
- Error message text: "Settings cannot be opened or changed in combat."

## Data Contract

### New DB field: `db.tip.stackColors`

```lua
-- DEFAULTS.tip addition in Core.lua:
stackColors = {
    [0] = { r = 1,       g = 1,       b = 1,       a = 1 }, -- white
    [1] = { r = 0.18039, g = 0.80000, b = 0.44314, a = 1 }, -- green
    [2] = { r = 1,       g = 0.94118, b = 0,       a = 1 }, -- gold
    [3] = { r = 1,       g = 0.29804, b = 0.18824, a = 1 }, -- red-orange
},
```

Keys are integers 0–3. Values use the same `{r, g, b, a}` table format as `fillColor` / `borderColor`.

### Removed DB field: `cfg.enabled`

- Remove from `DEFAULTS.tip` in Core.lua.
- Remove from `NormalizeDB` validation.
- Remove all `cfg.enabled` checks in TipOfTheSpear.lua visibility logic — tracker visibility gated solely on `Tip.isSurvival`.

### `Tip:Update()` stack color read

```lua
-- Replace hardcoded STACK_COLORS[stacks] reads with:
local stackColors = GetCfg().stackColors or STACK_COLORS  -- STACK_COLORS as fallback only
local color = stackColors[stacks] or stackColors[0]
```

The local `STACK_COLORS` constant in TipOfTheSpear.lua remains as the fallback only — not the primary source during normal operation.

## Visibility / Show-Hide Contract

Control which sections are visible based on `cfg.displayMode`. Implemented in `Options:Refresh()`.

```
if cfg.displayMode == "bar" then
    barSection:Show()
    numberSection:Hide()
    window:SetSize(386, 380)   -- bar mode height
else  -- "number"
    barSection:Hide()
    numberSection:Show()
    window:SetSize(386, 484)   -- number mode height
end
```

Section containers (`barSection`, `numberSection`) are created as sub-frames of the window at `BuildWindow` time using `CreateFrame("Frame", nil, window)`. Each section frame has `SetSize(386, sectionHeight)` and all mode-specific widgets are parented to their respective section frame.
