<objective>
Establish the data-layer foundation for user-configurable per-stack number colors (DISP-06). Add a `colorByStack` boolean and a nested `stackColors` sub-table (keys 0-3) to `DEFAULTS.tip` in Core.lua so the existing `MergeDefaults` path populates them on fresh and legacy DBs with no settings wipe. Then replace the hardcoded `STACK_COLORS` read in the number-mode render path of TipOfTheSpear.lua with a config read: toggle ON reads the per-stack config color; toggle OFF falls back to the single flat `textColor` (mirroring the existing RefreshLayout fallback).

Purpose: The Options.lua widgets (plan 02) and the new spec coverage (plan 03) both depend on these config fields and this render behavior existing first. This is the sole data-layer plan; keep the two source files in this wave.
Output: New `DEFAULTS.tip.colorByStack` + `DEFAULTS.tip.stackColors` fields; config-driven number-mode color rendering.
</objective>
