<objective>
Remove the `icons` display mode in full from the three addon source files so the addon ships exactly two modes: `bar` and `number`. This is a pure removal/refactor of well-understood existing code — no new modes, libraries, or WoW APIs.

Scope: delete the `icons` rendering branches in `TipOfTheSpear.lua` (`RefreshLayout` and `Update`); remove the `icons` token, the legacy `icon`->`icons` alias, and the `icons` validation entry in `Core.lua`; remove the "Icons" mode button and the `icons` `MODE_LABELS` entry in `Options.lua`; remove the now-orphaned `iconSize`/`iconSpacing` `DEFAULTS` and their two Options sliders; update `/dmax` help text to `bar|number`.

Implements the LOCKED Phase 5 decisions (user-confirmed 2026-06-23): final mode set is exactly `bar` + `number`; NO `bartext` mode (the 2026-06-22 bartext decision is REVERSED); NO migration path for persisted `icons`/`icon` values — validation simply falls back to default `bar`.

Purpose: A smaller, simpler mode surface with no dead rendering path or orphaned config.
Output: Three modified source files; zero `icons`/`iconSize`/`iconSpacing`/`bartext` hits in `Duncedmaxxing/`.
</objective>
